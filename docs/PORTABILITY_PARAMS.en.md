# Portability Parameters (Checklist)

Japanese version: [PORTABILITY_PARAMS_JA.md](./PORTABILITY_PARAMS_JA.md)

This checklist lists environment-dependent parameters required when moving `amazon-notify` to another host.

## 1. Environment-dependent values

| Category | Parameter | Example | Where to set | Notes |
|---|---|---|---|---|
| Filesystem | `BASE_DIR` | `/opt/amazon-notify` | `install-systemd.sh --base-dir` | code directory |
| Linux user | `SYSTEM_USER` | e.g. `ubuntu`, `deploy` | `install-systemd.sh --system-user` | reflected in systemd `User=` |
| Config path | `CONFIG_PATH` | `/opt/amazon-notify/config.json` | `install-systemd.sh --config-path` | used by `ExecStart --config` |
| Runtime-dir anchored artifacts | dedupe/index files | `.discord_dedupe_state.json`, `*.index.json` | resolved from config directory | destination changes when `--config` changes |
| Heartbeat path | `HEARTBEAT_PATH` | `/opt/amazon-notify/runtime/pubsub-heartbeat.txt` | `install-systemd.sh --heartbeat-path` | used by pubsub + fallback |
| GCP | `PROJECT_ID` | `my-gcp-project` | `config.json`, CLI args | parent of topic/subscription |
| Pub/Sub | `TOPIC_ID` | `amazon-notify-topic` | `--setup-watch --pubsub-topic` | `projects/<PROJECT_ID>/topics/<TOPIC_ID>` |
| Pub/Sub | `SUBSCRIPTION_ID` | `amazon-notify-sub` | `config.json.pubsub_subscription` | `projects/<PROJECT_ID>/subscriptions/<SUBSCRIPTION_ID>` |
| Notification | `DISCORD_WEBHOOK_URL` | `https://discord.com/api/webhooks/...` | `config.json.discord_webhook_url` | operational value, keep secret |
| OAuth | `credentials.json` | `<base-dir>/credentials.json` | local file | Gmail API OAuth client |
| OAuth | `token.json` | `<base-dir>/token.json` | generated by `amazon-notify --reauth` | Gmail runtime token |
| ADC | Application Default Credentials | `~/.config/gcloud/application_default_credentials.json` | `gcloud auth application-default login` | required by Pub/Sub client |
| Failure alert | `DISCORD_ALERT_WEBHOOK_URL` | `https://discord.com/api/webhooks/...` | `<base-dir>/deployment/systemd/amazon-notify-alert.env` | optional OnFailure alert |

## 2. Minimum values to replace

1. `PROJECT_ID`
2. `TOPIC_ID` / `SUBSCRIPTION_ID`
3. `BASE_DIR`
4. `SYSTEM_USER`
5. `DISCORD_WEBHOOK_URL`

## 3. Mapping points

`config.json` example:

```json
{
  "discord_webhook_url": "https://discord.com/api/webhooks/REPLACE_ME",
  "pubsub_subscription": "projects/PROJECT_ID/subscriptions/SUBSCRIPTION_ID",
  "pubsub_main_service_name": "amazon-notify-pubsub.service",
  "pubsub_heartbeat_file": "runtime/pubsub-heartbeat.txt"
}
```

systemd install:

```bash
sudo bash deployment/systemd/install-systemd.sh \
  --mode hybrid \
  --base-dir /path/to/amazon-notify \
  --system-user your_user \
  --config-path /path/to/amazon-notify/config.json \
  --heartbeat-path /path/to/amazon-notify/runtime/pubsub-heartbeat.txt
```

## 4. GCP checks

```bash
gcloud config set project PROJECT_ID

gcloud pubsub topics list --format='value(name)' | grep "projects/PROJECT_ID/topics/TOPIC_ID"
gcloud pubsub subscriptions list --format='value(name)' | grep "projects/PROJECT_ID/subscriptions/SUBSCRIPTION_ID"
```

Gmail watch IAM:

```bash
gcloud pubsub topics add-iam-policy-binding TOPIC_ID \
  --member="serviceAccount:gmail-api-push@system.gserviceaccount.com" \
  --role="roles/pubsub.publisher"
```

## 5. Pre-run validation

```bash
python3 -m json.tool ./config.json >/dev/null
amazon-notify --config ./config.json --validate-config
amazon-notify --config ./config.json --health-check
```

## 6. Common mistakes

- saving `config.json` as concatenated objects (`{...}{...}`)
- assuming `gcloud auth login` also sets ADC
- starting systemd units with unresolved placeholders (`YOUR_USER`, `/opt/amazon-notify`)
- project mismatch between topic/subscription and `config.json`
- switching `--config` but inspecting dedupe/index artifacts from the previous runtime directory
